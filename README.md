#fraud-monitoring-system
Project Overview

This project implements a fraud detection service built around a machine learning model. The goal of the system is to simulate how a fraud detection model would operate in a real production environment.

Instead of focusing only on training a model, this project demonstrates the full lifecycle of a machine learning system. The model is exposed through a REST API, containerized using Docker, deployed through CI/CD pipelines, and monitored for potential data drift.

Users can send transaction data to an API endpoint and receive a fraud prediction in real time. The system also keeps track of prediction usage and provides basic monitoring information.

Problem Motivation

Fraud detection is a common application of machine learning in financial systems. In real-world environments, the challenge is not only building the model but also deploying and maintaining it reliably.

A production system typically needs to:

Serve predictions through an API

Track how the model is being used

Monitor model behavior over time

Detect when incoming data patterns change

This project focuses on implementing these operational aspects.

API Endpoints

The service exposes several REST endpoints that allow interaction with the model.

Health Check
GET /health

Used to verify that the API service is running.

Example response:

{
  "status": "ok"
}
Model Information
GET /model-info

Returns basic information about the deployed model.

Example response:

{
  "model_name": "fraud_detection_model",
  "version": "1.0"
}
Fraud Prediction
POST /predict

Example request:

{
  "features": [0.15, 0.72, 0.03, 0.91]
}

Example response:

{
  "fraud_probability": 0.04,
  "threshold": 0.5,
  "prediction": 0
}

Where:

fraud_probability represents the probability of fraud

threshold is the classification threshold

prediction is the final result (0 = legitimate, 1 = fraud)

Statistics Endpoint
GET /stats

Example response:

{
  "total_predictions": 124
}
Reset Statistics
POST /reset-stats

Resets prediction counters.

Dashboard Endpoint
GET /dashboard

Provides monitoring information related to the system.

Running the Project Locally

Install dependencies:

pip install -r requirements.txt

Start the API server:

uvicorn app.main:app --host 0.0.0.0 --port 8080

Open API docs:

http://localhost:8080/docs
Running with Docker

Build the Docker image:

docker build -t fraud-monitoring-system .

Run the container:

docker run -p 8080:8080 fraud-monitoring-system

Then open:

http://localhost:8080/docs
CI/CD Pipeline

This project uses GitHub Actions to automate testing, deployment, and monitoring.

CI Workflow

Runs when code is pushed.

It:

installs dependencies

runs tests

validates the project

Deployment Workflow

Builds the Docker image and deploys the service to Google Cloud Run.

Monitoring Workflow

Runs periodically and checks for data drift using monitoring scripts.

Monitoring and Drift Detection

Machine learning models can lose accuracy if incoming data changes over time. This is known as data drift.

The monitoring workflow runs a drift detection script that analyzes incoming data and generates reports. These reports help determine whether the model should be retrained.
